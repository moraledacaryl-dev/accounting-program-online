# Pass 7 — Setup and Administration

This pass turns configuration work into one coherent administrative workflow rather than a collection of disconnected setup pages.

## Implemented

- Persistent Setup & Administration navigation across room types, rooms, rate plans, package rules, booking channels, chart of accounts, posting rules, master data, taxonomies, users, roles and permissions, Beds24, and system settings.
- Direct administrative actions for room management and system settings.
- Clear active-section treatment, including compatibility with legacy room setup and channel routes.
- Sticky headers for configuration tables and stronger scan hierarchy for high-density administrative records.
- Safer mobile table containment and horizontally scrollable setup navigation.
- Improved form scroll targeting beneath the sticky application and contextual navigation.
- Existing configuration behavior, permissions, APIs, imports, integrations, and accounting logic remain unchanged.

## Workflow ownership

Room and rate configuration defines sellable hotel inventory. Booking channels and Beds24 control distribution. Chart of accounts and posting rules determine accounting treatment. Master data and taxonomies govern reusable reference values. Users and roles define access. System settings remain the final administrative control surface.
