# UI Pass 3 — Hotel Operations

This pass changes the hotel-facing experience from a collection of independent CRUD pages into one navigable operating area.

## Delivered

- Action-first dashboard centered on arrivals, departures, occupancy, guest balances, daily cash close, and connected-app review.
- Removal of the duplicate dashboard workspace directory.
- Canonical links to Review Inbox, Inventory Items, Payroll Periods, and Restaurant Operations.
- Persistent Hotel Operations context navigation across Bookings, Calendar, Guests, Guest Folios, Events, Channel Payouts, and Rooms & Rates.
- Compact hotel page headers and reduced first-screen chrome.
- Clearer secondary treatment for create/edit forms.
- Sticky and more legible operational table headers.
- Responsive hotel navigation and safer narrow-screen presentation.
- Mobile dashboard movement rows reorganized for guest, room, status, and balance scanning.
- Secondary management widgets collapsed below operational content.

## Scope boundary

The existing booking, guest, folio, event, payout, room, rate, and channel APIs remain unchanged. Deeper route-specific lifecycle reconstruction continues through the remaining module passes and final QA.
